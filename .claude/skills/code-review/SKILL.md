# Code Review

コードレビューを体系的かつ建設的に行う。

## レビュー観点

以下の優先順で確認する:

1. **正確性**: ロジックにバグや意図しない動作がないか
2. **セキュリティ**: インジェクション、認証漏れ、機密情報のハードコード
3. **パフォーマンス**: N+1クエリ、不要なループ、メモリリーク
4. **可読性**: 命名、関数の粒度、コメントの適切さ
5. **テスタビリティ**: テストしやすい設計か、エッジケースの考慮

## 出力フォーマット

指摘は以下の形式で整理する:

- 🔴 **Critical**: 修正必須（バグ、セキュリティ）
- 🟡 **Warning**: 強く推奨（パフォーマンス、設計上の問題）
- 🟢 **Suggestion**: 改善提案（可読性、慣習）
- 💡 **Note**: 良い点やFYI

各指摘には対象箇所、理由、改善案を含める。

## 原則

- 批判ではなく改善提案として伝える
- 良い点にも触れる（ポジティブフィードバック）
- 「なぜ」を説明する（単なるスタイル好みは押し付けない）
- プロジェクトの規約があればそれに従う
